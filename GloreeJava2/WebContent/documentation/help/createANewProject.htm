Synopsis

 

This document describes how you can create a New Project in TraceCloud

Use Case

 

When you are starting a new Project, you may want it to have its own separate work space, user access rules and requirement types. TraceCloud makes it easy for you to create your own project. You can either create a brand new project or choose to clone a previous project. ‘Cloning’ a project makes sense if your process hasn’t changed since your last TraceCloud project, and you are happy with the way the previous project was set up. When cloning, you have the option of cloning only the

·         Clone Project Structure (Requirement Types, Folders, Roles and Permissions)

·         Clone Users and Project Structure

·         Clone Requirements, Users and Project Structure

·         Clone Traceability, Requirements, Users and Project Structure

Create a Brand New Project

 

To create a brand new project or to clone a project, log into TraceCloud and click the link to ‘Create New Project’ in the ‘My Projects’ tab.

 

In the next screen, go to the ‘Create New Project’ section and enter the project information

Notes:

·         Restricted Domains: This is useful if you want to restrict all members of a project to a certain email domain. For example, if you are an Oracle.com employee and are setting up a project for Oracle and want only users with oracle.com email domain ever having access to this project, you can set the restricted domain to oracle.com. Later on, when you or other administrators are adding users to this project, the system will ensure that only users with emails ending in oracle.com can be added to this project.

·         Administrators: Ensure that you are one of the administrators. Feel free to add others to this comma separated list.

·         Users: Feel free to add others to this comma separated list.  If these users are members of TraceCloud, they will have immediate access. If they are not, then they will receive an email with directions on how to create an account.

·         Please note that Creating a new Project is  a complex process, so give TraceCloud a few minutes and your new Project will show up in your ‘User Dashboard’

·         Once the project is created, if you would like to alter the structure or perform other Administrative tasks, you will need to log into the project, and go the ‘Administrator’ section of the project.

 

 

Clone an Existing Project

 

If you have already used TraceCloud and have refined your process (Requirement Types, User Roles, Access Rules, etc...) and would like your next project to have the same structure, you can choose to Clone this project.

 

To Clone an existing project, log into TraceCloud and click the link to ‘Create New Project’ in the ‘My Projects’ tab.

 

In the next screen, go to the ‘Clone an Existing Project’ section and …

·         From the drop down list select the project you want to clone

·         If you want to clone only the structure (Req Types, Folders, Roles and Permissions), choose the first check box

·         If you want to clones Users along with Structure, check the second box

·         If you want to clone the Requirements along with Users and Structure, use the third box

·         If you want to clone Traceability along with Requirements, Users and Structure then use the last box.

·         Please note that Cloning is  a complex process, so give TraceCloud a few minutes and your new Project will show up in your ‘User Dashboard’

·         Once the project is created, if you would like to alter the structure or perform other Administrative tasks, you will need to log into the project, and go the ‘Administrator’ section of the project.
