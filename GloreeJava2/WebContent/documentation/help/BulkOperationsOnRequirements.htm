Use Case : The user would like to run a complex report (Egg : Get All my Business requirements, that meet multiple filter criteria), and on all the requirements returned by this report, run some common operations (e.g. : Move to a new folder, Copy to a different req type, Delete, Purge, Set Traceability, Global search / replace text, Approve , Reject etc..)

 

Solution:

1.      Open your project

 

a.      Select the folder you want to bulk edit

b.      Report à Create A report

 

 

 

2.      Run report by

a.      Apply the Step 1: Filters. Usually the ‘Include Requirements in Sub folder’ is important, and so is apply any filters on attributes you are trying to bulk edit

b.      In Step 2, definitely display the attributes you are trying to bulk edit. Remember Ctrl+Click to select more than one field. Or use Mouse click and drag

c.      Remember to set the ‘Rows per Page’ to be sufficiently large, because bulk edit applies only to the requirements that you select in the check box. More on this when we get to image 3.

 

 

3.      Run Report

4.      When the report comes back, a few things to note

a.      You can always click on ‘Show filter’ to show the filter and change your report criteria

b.      For requirements you want to bulk edit, you need to check the box next to them. One way to select a lot of requirements is to use the ‘Select’ column header.

c.      If you have multiple pages of data, you may have to bulk edit for each page. So here is the secret sauce that works for me

                                                    i.     I play with my filters, display attributes,  and ‘rows per page’ parameter, till I get the exact data set I want to bulk edit, in one page, then I click the ‘Select’ to check all the boxes and then do the bulk edit.

 

 

 

5.      On this page, you will see Standard Actions, Custom Actions, Traceability Actions and Approval Actions.

 

6.      If you plan to do this often, then you do some magic here.

a.      In Show Filters, you can see a link called ‘Save Reports’. Give it a name and save it.

b.      At the top of your project page, you will see ‘Dashboards’. Click on ‘My Dashboards’ and this report will show up as your saved reports.

c.      From that point onwards it will be easy to do bulk edit.

7.      To keep an eye on this report automatically, you can have the TraceCloud system run the report at regular basis and email it to you.
